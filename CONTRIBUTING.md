# CONTRIBUTING.md

## English

Thanks for helping improve Praxis.

Before opening a pull request:

1. Read `AGENT_GUIDE.md`.
2. Run the relevant tests, and run the Release configuration test when feasible.
3. Update docs and `CHANGELOG.md` when behavior changes.
4. Do not create tags or publish packages.
5. Use explicit `git add <file>` paths only.

If you are reporting a bug, include the exact steps inside Praxis (button setup, command, key sequence), the Praxis app version, OS, and sanitized output.
Do not paste private paths, customer data, proprietary binaries, or exploit details into public issues.

## 日本語

Praxis の改善に協力していただきありがとうございます。

プルリクエストを出す前に次を確認してください。

1. `AGENT_GUIDE.md` を読む。
2. 関連テストを実行し、可能なら Release 構成のテストも走らせる。
3. 挙動が変わった場合はドキュメントと `CHANGELOG.md` を更新する。
4. タグ作成やパッケージ公開はしない。
5. `git add <file>` で明示的に追加する。

不具合報告には、Praxis 内での具体的手順（ボタン構成、コマンド、キー操作）、Praxis アプリのバージョン、OS、匿名化した出力を含めてください。
public issue に private なパス、顧客データ、プロプライエタリなバイナリ、攻撃詳細を貼らないでください。
