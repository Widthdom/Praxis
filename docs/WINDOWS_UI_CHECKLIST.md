# Windows UI 確認チェックリスト

Avalonia 版 Praxis を Windows で確認するときは、このセッションで詰めた仕様が退行していないかを次の観点で確認する。

## ウィンドウとクローム

- [ ] 最小化、最大化/復元、閉じるボタンが Windows 標準どおり右上に並ぶ。
- [ ] 最大化ボタンは通常の Windows 最大化であり、macOS のような FullScreen には入らない。
- [ ] 最大化/復元を繰り返してもメインコンテンツの余白、角丸、背景色が崩れない。
- [ ] 閉じる、最小化、最大化/復元のクリック範囲が十分に広く、リサイズ判定に奪われない。
- [ ] カスタムタイトルバー全体をドラッグできる。特に最上端付近でも移動を開始できる。
- [ ] Command/Search 欄やボタン上ではタイトルバー移動が誤発火しない。

## リサイズ

- [ ] 上下左右辺でリサイズできる。
- [ ] 四隅でリサイズできる。
- [ ] つかんだ辺/角に合ったリサイズカーソルが出る。
- [ ] リサイズ中にカーソルが arrow に戻らない。
- [ ] 四隅のリサイズ判定が広すぎず、キャプションボタン操作を邪魔しない。
- [ ] 角リサイズ時に画面がちらつかず、連続的にサイズが変わる。
- [ ] 最小サイズ未満に縮まず、最小サイズ付近でもレイアウトが破綻しない。

## メイン背景と擬似アクリル

- [ ] メインウィンドウ外周が角丸に見える。
- [ ] ウィンドウ角に余計な背景や白/黒い余白が見えない。
- [ ] ダークモードの擬似アクリルは強すぎず、背景が少し透けて見える。
- [ ] ライトモードの擬似アクリルも単なる半透明ではなく、すりガラス感が視認できる。
- [ ] モーダルや Edit/Delete 背面の覆いが画面端まで広がる。
- [ ] 背面覆いもライト/ダーク双方で擬似アクリルとして自然に見える。

## Command/Search 欄

- [ ] Command 欄と Search 欄の上端位置、高さ、角丸が揃っている。
- [ ] フォーカス前後で文字位置、キャレット位置、テキストボックスの高さがずれない。
- [ ] フォーカス枠は青ではなくモノトーンで表示される。
- [ ] 日本語入力中に文字化けしない。
- [ ] placeholder アイコンは未入力かつ未フォーカス時だけ表示される。
- [ ] 入力開始後、未確定の日本語変換中でも placeholder アイコンが邪魔しない。
- [ ] Search 欄の虫眼鏡アイコンは左寄せで、柄が丸を突き抜けない。
- [ ] Command/Search 欄の丸いクリアボタンは、丸とバツの中心が一致する。
- [ ] クリアボタンは Tab stop にならない。
- [ ] クリアボタンの配色はライト/ダークで対になり、背景に溶け込まない。
- [ ] クリアボタン表示時にテキストボックス内に余計なスクロールバーが出ない。

## 新規追加ボタン

- [ ] 新規追加ボタンの高さは `40` で、Command/Search 欄と視覚的に揃う。
- [ ] アイコンは既存 SVG の思想に沿った、円に内接/外接する正六角形と中央のプラス記号である。
- [ ] アイコンは `P` ではない。
- [ ] 通常、hover、pressed の色がボタン配置領域のボタンと揃う。
- [ ] ライトモードでアイコンが白すぎず、背景に埋もれない。
- [ ] ダークモードで hover したとき、通常時との差が視認できる。
- [ ] クリックで新規作成モーダルが開き、ButtonText が全選択される。

## Command 候補

- [ ] Command 欄で下矢印を押すと候補選択が始まる。
- [ ] 上下矢印で候補間を移動できる。
- [ ] 初期状態で未選択なのに勝手に強調表示されない。
- [ ] 選択行と hover 行は候補領域の全幅で塗られる。
- [ ] 選択行と hover 行の高さが揃っている。
- [ ] ダークでは hover が通常より明るい灰色、ライトでは通常より暗い灰色になる。
- [ ] Command、ButtonText、Arguments の列位置が揃う。
- [ ] 候補領域の幅はボタン配置領域と揃う。
- [ ] 候補領域は配置領域の上に重なって開き、Search 欄の幅を押し広げない。
- [ ] 候補を右クリックするとボタン配置領域と同じ context 動作になる。
- [ ] 候補を中央クリックすると編集動作になる。

## ボタン配置領域

- [ ] 配置領域の土台は透明で、不要な枠が見えない。
- [ ] 起動時に見切れていたボタンも、スクロールやウィンドウ拡大で描画される。
- [ ] viewport 周辺のボタンだけが描画対象になり、大量ボタン時もスクロールが重くなりにくい。
- [ ] ボタンをドラッグ移動できる。
- [ ] ドラッグ中のカーソルは通常 arrow のまま変わらない。
- [ ] ドラッグ移動のたびに不要なステータス文言が出ない。
- [ ] Ctrl+クリックで選択状態を切り替えられる。
- [ ] 矩形選択で対象ボタンを選択できる。
- [ ] 選択矩形は緑などのアクセント色ではなくモノトーンで表示される。
- [ ] 右クリックで Edit/Delete メニューが開く。
- [ ] 中央クリックで編集が開く。
- [ ] ボタンは Tab stop にならない。
- [ ] hover tooltip が表示される。
- [ ] tooltip の見出しは太字で、値が折り返されすぎない。
- [ ] invert theme が有効なボタンは見た目が反転する。

## Dock

- [ ] Dock は水平スクロールできる。
- [ ] Dock ボタンの幅が揃う。
- [ ] Dock ボタンのデザインが配置領域のボタンと揃う。
- [ ] Dock ボタンの下端が水平スクロールバーに削られない。
- [ ] スクロールバーが hover で太くなってもボタンと重ならない。
- [ ] Dock ボタンは配置領域の選択状態を引き継がない。
- [ ] Dock ボタンは hover 時だけ色が変わる。
- [ ] Dock ボタンは Tab stop にならない。
- [ ] 右クリック、中央クリックが配置領域のボタンと同じ意味で動く。

## モーダルエディタ

- [ ] モーダルにタイトルは表示されない。
- [ ] 上から GUID、ButtonText、Tool、Arguments、Clip Word、Note、Invert Theme の順に並ぶ。
- [ ] `Note Word` ではなく `Note` と表示される。
- [ ] 見出しとテキストボックスは高さ方向に中央揃えされる。
- [ ] 見出しとテキストボックスの距離が離れすぎない。
- [ ] GUID は編集不可である。
- [ ] 新規追加で開いた場合、ButtonText が全選択される。
- [ ] 既存ボタン編集で開いた場合、ButtonText の末尾にキャレットが置かれる。
- [ ] Tab 順は ButtonText、Tool、Arguments、Clip Word、Note、Cancel、Save、GUID の順に循環する。
- [ ] GUID で Shift+Tab すると Save に戻る。
- [ ] モーダル外をクリックしてから Tab を押しても、メイン画面側へフォーカスが移らない。
- [ ] Esc は Cancel と同じ動作になる。
- [ ] Ctrl+S は Save と同じ動作になる。
- [ ] Cancel/Save は中央寄せで横に並ぶ。
- [ ] Cancel/Save は選択状態だけで背景色が変わり、hover/pressed で別色に変わらない。
- [ ] モーダルのテキストボックスはフォーカスしても背景色が変わらない。
- [ ] モーダルのテキストボックスのフォーカス枠は青ではなくモノトーン。
- [ ] Clip Word/Note は基本 1 行の高さから始まり、改行に応じて一定上限まで伸びる。
- [ ] Clip Word/Note が長くなってもモーダル全体が画面外へ伸びず、内部スクロールになる。
- [ ] テキストボックス右端のクリップボタンは、右端 hover 時だけ表示される。
- [ ] クリップ対象が空文字のとき、クリップボタンは hover しても表示されない。
- [ ] クリップボタンは背景なしで、アイコンだけが控えめに見える。
- [ ] クリップボタンアイコンはライト/ダーク双方で背景に同化しない。
- [ ] チェックボックスは青を使わずモノトーンで、ライト/ダークが対になっている。
- [ ] チェックボックス未選択時は枠なし塗りつぶし、選択時は文字色と同系のチェックマークが出る。

## Edit/Delete メニューとテキストメニュー

- [ ] Edit/Delete メニュー表示時、背面に全画面の擬似アクリル覆いが出る。
- [ ] 開いた直後は Edit が選択状態になる。
- [ ] 非選択側には不要な塗りが出ない。
- [ ] Tab と上下キーで Edit/Delete を移動できる。
- [ ] hover だけでは選択色が変わらない。
- [ ] 選択中かつ hover の場合は選択色が維持される。
- [ ] テキストボックス右クリックメニューは Edit/Delete と同系の小さなデザインで表示される。
- [ ] テキストボックス右クリックメニューにショートカットキー表示が出ない。
- [ ] メニュー自体と選択矩形に角丸があり、左右の余白がある。

## コンフリクトダイアログと同期

- [ ] 別ウィンドウでの DB 更新を検知して同期する。
- [ ] 編集中に外部更新が入った場合、コンフリクトダイアログが表示される。
- [ ] コンフリクトダイアログの背面にも全画面の擬似アクリル覆いが出る。
- [ ] Reload、Overwrite、Cancel がエディタモーダルと同じ見た目で並ぶ。
- [ ] Reload、Overwrite、Cancel のフォーカス/Tab/hover/pressed 挙動がエディタモーダルと揃う。
- [ ] Reload は外部変更を取り込み、Overwrite は現在の編集内容を保存し、Cancel は何もせず閉じる。

## クリップ通知

- [ ] クリップボタン押下後、画面中央に小さな角丸通知が一瞬出る。
- [ ] 通知アイコンは通知ボックスの中央に配置される。
- [ ] ダークモードでは半透明の濃色背景、枠、見えるアイコンで表示される。
- [ ] ライトモードでもダークと対称に、半透明背景、枠、見えるアイコンで表示される。
- [ ] 通知は短時間でフェードアウトし、操作を邪魔しない。

## ステータスバー

- [ ] ステータスバーの表示領域は最初から確保され、表示時に Dock が上へずれない。
- [ ] ステータスバーは角丸で表示される。
- [ ] Ready/idle はニュートラルグレー。
- [ ] 成功は緑、処理中は黄、エラーは赤で表示される。
- [ ] 成功/処理中/エラーはグレースケール対象外として色を使う。
- [ ] ステータス文言は一定時間後にフェードアウトする。
- [ ] ボタン移動だけでは `Button moved` のような不要なステータスを出さない。

## テーマ切り替え

- [ ] Ctrl+L、Ctrl+D、Ctrl+H でライト、ダーク、システムテーマを切り替えられる。
- [ ] ライトからダーク、ダークからライトへ戻しても色が中途半端に残らない。
- [ ] Command/Search、モーダル、クリアボタン、チェックボックス、通知、ステータスバーがテーマに追従する。
- [ ] ライト通常色とダーク invert 色、ダーク通常色とライト invert 色の対応が破綻しない。
- [ ] 灰色系 UI は RGB の偏りがなく、明暗差で表現される。
- [ ] ボタン通常色、hover 色、選択色が背景に溶け込まない。

## キーボードとショートカット

- [ ] Ctrl+Z で Undo が走る。
- [ ] Ctrl+Y または Ctrl+Shift+Z で Redo が走る。
- [ ] モーダル中の Esc/Ctrl+S がメイン画面側に漏れない。
- [ ] ボタン配置領域ボタン、Dock ボタン、新規追加ボタンの Tab stop 方針が仕様どおりである。
- [ ] コンテキストメニューやモーダル表示中にキャプションボタンへ Tab フォーカスが逃げない。

## Windows 固有の最終確認

- [ ] Windows 最大化時、タスクバーを覆わない。
- [ ] Windows 最大化/復元時、角丸と影が不自然に残らない。
- [ ] Windows の DPI スケール変更時、テキスト、ボタン、スクロールバーが重ならない。
- [ ] Windows のライト/ダーク設定に追従する System テーマが動く。
- [ ] タスクバーに表示されるアプリアイコンが v1 と同じ既存アイコンになる。
